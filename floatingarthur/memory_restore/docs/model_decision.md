# AI 복원 모델 결정

작성일: 2026-07-22

## 결정

| 역할 | 선택 | 적용 범위 | 기본값 |
| --- | --- | --- | --- |
| 전체 사진 초해상도·노이즈 완화 | Real-ESRGAN `RealESRGAN_x4plus` | 역사자료·개인사진 | 선택 시 2배 확대 |
| 얼굴 복원 | GFPGAN v1.3 | 개인사진만, 별도 동의 시 | 기본 비활성 |
| 기준선 비교 | Pillow 보정 | 모든 입력 | 항상 제공 |

## Real-ESRGAN을 주 모델로 고른 이유

1. 일반 사진·압축 손상·노이즈를 대상으로 한 실용적 이미지 복원 모델이며, 공식 저장소가 일반 이미지 추론과 타일 처리·임의 확대 배율을 안내한다.
2. 주 시연인 을지로 역사자료는 인물보다 지하 통로·간판·공간 디테일이 핵심이다. 얼굴 전용 모델을 기본으로 쓰는 것보다 목적에 맞다.
3. BSD-3-Clause 라이선스여서 공모전 프로토타입에 출처와 라이선스를 남기며 사용하기 용이하다.

출처: [Real-ESRGAN 공식 저장소](https://github.com/xinntao/Real-ESRGAN), [BSD-3-Clause 라이선스](https://github.com/xinntao/Real-ESRGAN/blob/master/LICENSE)

## GFPGAN을 기본 경로에서 제외하는 이유

GFPGAN은 얼굴 전용 복원 모델이다. 개인사진에서는 선택지가 될 수 있지만, 역사사진의 사람 얼굴이나 표정을 그럴듯하게 추정해 사실처럼 보이게 할 위험이 있다. 따라서 기본 비활성으로 두고, 개인사진에서만 별도 고지·동의를 거쳐 사용한다.

출처: [GFPGAN 공식 저장소](https://github.com/TencentARC/GFPGAN), [Apache-2.0 라이선스](https://github.com/TencentARC/GFPGAN/blob/master/LICENSE)

## 서비스 표시 원칙

- `보존형`: 원본 사본만 제공. AI 미사용.
- `보수형`: 기준선 보정 또는 Real-ESRGAN 2배 확대. 사용 모델·배율·타일 설정·결과 시간을 기록.
- `표현형`: 강한 보정 또는 AI 결과. “AI가 추정한 세부 묘사가 포함될 수 있음”을 표시.
- 역사사진은 색채화·인물 애니메이션·자동 장소 단정을 제공하지 않음.

## 서버 연결 방식

Flask는 GPU 라이브러리를 직접 알아야 하지 않는다. `REALESRGAN_COMMAND`에 설정된 학교 서버 실행기를 호출하고, 프론트엔드는 복원기록 API만 받는다. 실행기가 없을 때는 명시적으로 `AIBackendUnavailable`을 반환하며, 조용히 기준선 결과를 AI 결과로 위장하지 않는다.
