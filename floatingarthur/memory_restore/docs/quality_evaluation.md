# 해상도 기반 모델 선택 및 정량평가

## 1. 입력 해상도별 기본 선택

| 입력 크기 | 기본 후보 | 출력 | 이유 | 검토 |
| --- | --- | --- | --- | --- |
| 0.5MP 미만 | `RealESRGAN_x4plus`와 기준선 비교 | 최대 2배 | 정보가 크게 부족해 AI 후보를 비교하되, 과장 위험이 큼 | 필수 |
| 0.5~2.0MP | `RealESRGAN_x2plus`와 기준선 비교 | 2배 | 과도한 재생성보다 중간 확대가 적절한 범위 | 필수 |
| 2.0MP 이상 | 보존형·Pillow 보수형 우선 | 원본 크기 | AI 확대가 오히려 문자·질감을 바꿀 수 있음 | 선택 |

이 규칙은 해상도만으로 AI를 강제하지 않는다. 각 단계에서 원본 보존형과 비교한 뒤 정량 기준을 통과한 결과만 기본 추천으로 제시한다.

## 2. 기술별 비교 대상

| 후보 | 역할 | 역사사진 기본 사용 | 비교 목적 |
| --- | --- | --- | --- |
| 원본 보존 | 무처리 | 예 | 역사적 근거 기준점 |
| Pillow 기준선 | 대비·노이즈·리사이즈 | 예 | AI가 아닌 최소 보정 기준 |
| RealESRGAN_x2plus | 중간 해상도 초해상도 | 조건부 | 0.5~2.0MP에서 AI 효과 검증 |
| RealESRGAN_x4plus | 저해상도 초해상도 | 조건부 | 0.5MP 미만에서 AI 효과 검증 |
| GFPGAN | 얼굴 복원 | 아니오 | 개인사진 별도 동의 시에만 비교 |

## 3. 정량 평가 방식

역사 원본에는 완전한 정답 이미지가 없으므로, 모델 성능의 ‘정답 비교’는 원본을 1/2 크기·JPEG 품질 35으로 의도적으로 열화한 통제 세트에서 수행한다.

| 지표 | 의미 | 채택 기준 |
| --- | --- | --- |
| PSNR(dB) | 원본 대비 픽셀 오차 | Pillow 기준선보다 +0.3dB 이상 또는 SSIM 조건 충족 |
| SSIM | 구조적 유사성 | Pillow 기준선보다 +0.01 이상 또는 PSNR 조건 충족 |
| 처리시간(ms) | 시연 지연 | 한 장당 30초 이내를 우선 목표 |
| Edge strength | 선명도 변화 관찰용 | 단독 채택 근거로 사용하지 않음 |
| 사람 검토 | 문자·얼굴·건축 디테일 왜곡 여부 | 역사자료에서 필수 |

PSNR·SSIM은 `scikit-image.metrics`의 검증된 구현으로 계산한다. 정량 우위가 있어도 간판 문자, 인물, 건축 구조가 바뀐 결과는 채택하지 않는다.

## 5. 시연 UI와의 분리

- 평가 산출물은 `benchmarks/{실험시각}/benchmark_results.csv`, `benchmark_report.json`에만 저장한다.
- Flask의 사용자용 API·복원기록 카드에는 PSNR, SSIM, 처리시간, 후보 모델 순위를 포함하지 않는다.
- 시연 화면에는 사용자가 선택한 결과, 원본 출처, 복원 강도, AI 사용 여부와 주의 문구만 표시한다.
- CSV는 추후 모델·해상도 구간을 추가해 재실험할 수 있도록 한 행이 한 후보 결과가 되게 유지한다.

## 4. 결과 해석 원칙

- 통제 세트의 점수는 ‘이 방식이 인위적으로 열화된 사진을 얼마나 복원하는가’를 보여 주며, 역사적 사실성을 증명하지 않는다.
- 실제 역사사진은 참조 정답이 없으므로, 점수 대신 복원기록·원본 비교·사람 검토로 최종 결과를 결정한다.
- UI에는 승자 모델만 숨겨 보여주지 않고 원본·보수형·AI 후보를 나란히 두고 사용자가 선택하게 한다.
