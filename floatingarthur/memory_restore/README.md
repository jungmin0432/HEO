# 을지로 기억복원소 - Flask 프로토타입

AI가 과거를 단정하지 않고, 원본 보존·복원 강도 선택·출처 기록을 통해 사용자가 기억을 해석하도록 돕는 시연용 웹앱이다.

## 현재 단계

- Flask 애플리케이션 골격과 장소 카드 API
- 역사 자료의 출처·현재 장소 매칭 상태 관리
- 원본 사진을 별도 `../photos` 폴더에 보관하는 읽기 전용 연결

아직 포함하지 않는 것: 실제 AI 복원 모델, 자동 장소 추정, 얼굴 인식, 결제·점포 주문 연동.

## 실행

```powershell
.\.venv\Scripts\python.exe app.py
```

이 저장소는 **화면을 포함하지 않는 Flask 백엔드**다. 다른 팀의 반응형 웹·앱은 이 서버의 JSON API와 이미지 URL만 사용한다. 화면 구현을 위한 HTML, CSS, JavaScript는 이 단계에서 만들지 않는다.

로컬 확인은 `http://127.0.0.1:5000/api/v1/places`에서 한다.

## 기준선 복원 결과 생성

```powershell
.\.venv\Scripts\python.exe -m scripts.build_sample
```

명령은 서울기록원 역사사진 한 장의 원본 사본, 보수형 보정, 표현형 보정을 `outputs/`에 생성한다. 이 단계는 생성형 AI가 아닌 이미지 처리 기준선이다. Pillow는 서버 내부 처리에만 쓰이며, 프론트엔드에 설치하거나 노출하지 않는다.

## 장소 매칭 규칙

- `EXACT`: 같은 랜드마크와 촬영 시점이 확인된 과거·현재 쌍
- `NEARBY`: 같은 을지로 생활권이지만 같은 시점은 확인되지 않은 쌍
- `REFERENCE`: 과거 사진의 맥락만 제공하며 현재 사진과 일대일 비교를 주장하지 않음

현재 내려받은 공공 자료는 동일 시점 검증 전까지 `REFERENCE`와 `NEARBY`만 사용한다. 모든 원본·가공본 화면에는 기록명, 기록번호, 연도, 출처 링크, 복원 설정을 표시하도록 후속 화면을 설계한다.
