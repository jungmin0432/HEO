## 과거·현재 매칭 원칙

1. 자동 장소 추정이나 얼굴 인식은 MVP에 넣지 않는다.
2. 사용자가 장소 카드를 직접 선택하고, 운영자가 검토한 현재 사진을 연결한다.
3. 페어마다 아래 중 하나를 화면에 명시한다.
   - `EXACT`: 동일 랜드마크와 시점이 확인된 쌍
   - `NEARBY`: 같은 을지로 생활권이지만 동일 시점 미확인
   - `REFERENCE`: 역사 참고 사진만 제공, 현재 사진과 일대일 비교하지 않음
4. 지금 내려받은 자료만으로는 `EXACT` 쌍을 확정하지 않는다. 1983년 을지로 2호선 개통 기록은 주 복원 입력으로 쓰고, 현재 이미지는 `NEARBY` 또는 `REFERENCE`로 표시한다.

## 구현 전 선별 기준

- 사진에 건물·출입구·역명·통로 구조처럼 현재와 대조할 단서가 있는가
- 훼손(스크래치·노이즈·저해상도)이 있어 복원 전후를 설명할 수 있는가
- 인물 프라이버시·초상권을 별도로 검토해야 하는 장면은 피할 수 있는가
- 출처·연도·장소를 서비스 화면에 짧고 정확하게 표시할 수 있는가
