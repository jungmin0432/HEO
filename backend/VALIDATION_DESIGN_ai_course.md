# AI 코스 설계기 — 모델 비교 사전 검증 설계 v2

작성 시각: 실행 전 커밋 (결과를 본 뒤 기준을 바꾸지 않기 위함)

## v2 변경 사유
1라운드(gpt-oss-120b/20b + llama-3.3-70b)에서 폴백율 지표가 모델별 무료
TPM 한도 차이로 교란됨을 확인. v2는 (a) 서로 다른 벤더/계열의 모델로 교체하고
(b) 호출 간격을 늘려 속도제한 교란을 줄이고 (c) "총 도보거리" 정량 지표를
추가한다.

## 비교 대상 (Groq 무료 티어, 후보 3개 — 전부 다른 벤더/계열)

| 모델 | 벤더 | 특징 |
| --- | --- | --- |
| `llama-3.3-70b-versatile` | Meta | 1라운드 최고 성능, 재확인용 유지 |
| `llama-3.1-8b-instant` | Meta | 같은 계열의 훨씬 작고 빠른 모델 |
| `qwen/qwen3.6-27b` | Alibaba | 다른 벤더, 중간 크기 |

## 측정 지표 (실행 전 정의)

1. **환각률**: LLM이 candidates 목록에 없는 id를 stop_ids에 넣은 비율 (`dropped_hallucinated` 존재 여부) — 0%가 목표
2. **예산위반율**: 검증기가 시간예산 초과로 stop을 제거해야 했던 비율 (`dropped_over_budget` 존재 여부) — 0%가 목표
3. **완전 폴백율**: 검증 후 남는 stop이 0개라 결정적 알고리즘으로 대체된 비율 — 낮을수록 좋음
4. **의도 반영 점수 (사람 채점, 0~2)**: 자유문장 속 버튼으로 표현 안 되는 뉘앙스(계단 회피, 아이 동반, 복합 목적 등)를 코스가 반영했는가
   - 0 = 무관한 코스
   - 1 = 표면적 목적(시간/purpose)만 반영, 뉘앙스는 놓침
   - 2 = 뉘앙스까지 반영
5. **응답 시간(초)**, **토큰 사용량**: API 응답의 실측치 그대로 기록

## 성공 기준 (실행 전 정의, v2)

1. 환각률 0% — 필수, 하나라도 어기면 그 모델은 탈락
2. 완전 폴백율 20% 미만 — 단, 폴백 사유가 429(속도제한)면 별도로 표시하고
   이 기준 판정에서는 제외 (모델 품질이 아니라 무료 한도 문제이므로)
3. 평균 의도 반영 점수 1.5/2.0 이상 (사람 채점)
4. 1~3을 통과한 후보 중 **평균 총 도보거리(m)가 가장 짧은 모델**을 최종 채택
   — 같은 조건이면 사용자가 덜 걷는 코스를 설계하는 모델이 더 낫다고 본다
5. 도보거리가 동률에 가까우면(5% 이내) 응답 속도가 빠른 쪽을 채택
